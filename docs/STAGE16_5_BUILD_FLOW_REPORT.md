# Stage 16.5 Build Flow Report

## Summary

Stage 16.5 hardens the player-facing path for the Stage 16 vertical slice without starting Stage 17. It fixes the exported-log root Play Mode error, hides debug presentation by default, adds a boot scene, and adds Windows player build/check scripts.

## Root Error

The exported project log first failed in:

```text
NullReferenceException
ProjectAegisRTS.UnityClient.UI.Desktop.ProductionCategoryTabs.BuildIfNeeded()
Assets/Rts/Scripts/UI/Desktop/ProductionCategoryTabs.cs:66
```

The repeated 999+ console errors cascaded from the same desktop HUD initialization path. The Stage 16 scene had editor-generated UI children, but the runtime-only `buttons` field was empty on Play. `BuildIfNeeded()` tried to add another `GridLayoutGroup` to `Production Tabs`, Unity rejected the duplicate layout group, and the next line dereferenced the failed component add.

The fix reuses existing layout/components and reconnects button listeners instead of rebuilding on top of editor-generated UI. Neighboring HUD builders with the same runtime-field pattern were hardened too.

## Player Presentation

`PlayerBuildSceneInitializer` now enforces the default camera framing:

- Position: `16, 38, -26`
- Rotation: `60, 0, 0`
- Orthographic: `true`
- Size: `28`
- Clip planes: `0.1 / 1000`

`DebugHudVisibilityController` hides debug/status panels by default while leaving their existing hotkeys intact. Placement UI is hidden unless placement mode is active. The objective HUD remains visible, but validation/debug action buttons are hidden by default.

## Boot Flow

`Assets/Rts/Scenes/Stage16_5_Boot.unity` is generated by:

```text
ProjectAegisRTS > Stage 16.5 > Configure Player Build Flow
```

Build Settings are configured with:

1. `Assets/Rts/Scenes/Stage16_5_Boot.unity`
2. `Assets/Rts/Scenes/Stage16_PlayableVerticalSlice.unity`

Earlier diagnostic/showcase scenes may remain after those entries for validation compatibility, but none appears before Boot.

## Validation

Use:

```powershell
.\tools\run-stage16-player-build-checks.ps1 -SkipPlayerBuild
.\tools\build-windows-player-stage16.ps1
```

`run-stage16-player-build-checks.ps1` runs core tests, Unity DLL build, medium recursion audit, Stage 16 medium validation, Stage 16.5 build-flow validation, Stage 16.5 smoke validation, UnityEngine-free scan, and `git diff --check`.

`build-windows-player-stage16.ps1` builds the Windows player into:

```text
build/windows-player-stage16/ProjectAegisRTS.exe
```

## Not Final

Stage 16.5 does not add multiplayer, replay/checksum systems, final art, final balance, campaign scripting, or release packaging.
